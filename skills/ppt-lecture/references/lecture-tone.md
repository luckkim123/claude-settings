# Lecture Tone — 강의 글쓰기 톤

학술 발표(`ppt-academic`)와 다른 강의/교육 자료 전용 톤 가이드.

---

## 학술 톤과의 차이

| 항목 | 학술 (`ppt-academic`) | 강의 (`ppt-lecture`) |
|:---|:---|:---|
| 동사 | "제안한다", "보인다" | "이해한다", "쓴다", "익힌다" |
| 시점 | 객관적 ("본 연구는") | 주관적 OK ("우리가 살펴볼") |
| 청중 호칭 | 명시 X | "여러분", "우리" 가능 |
| 1인칭 | 사용 X | OK ("제가", "저희가") |
| 문체 | 격식 ("~한다") | 캐주얼 OK ("~하다", "~해요") |
| 약어 | 첫 등장 시 풀네임 + 약어 | 일반 약어는 풀네임 생략 가능 |

---

## 동사 / 시점

**선호**:
- "이해한다", "익힌다", "기억한다"
- "예를 들어", "구체적으로 말하면"
- "이게 왜 중요하냐면", "여기서 핵심은"

**회피**:
- 너무 격식 있는 학술 동사 ("규명한다", "고찰한다")
- 모호한 표현 ("어느 정도", "비교적")

---

## 슬라이드 글쓰기

### 제목

**선호**:
- 의문형 OK ("왜 RNN이 졌는가")
- 직관적 ("3단계로 끝")
- 짧고 강력 ("Self-Attention")

**회피**:
- 명사형 종결 강제 X (학술과 달리 동사형 OK)
- 너무 추상적 ("개념 이해")

**예시**:
- ❌ "Self-Attention의 분석"
- ✅ "Self-Attention" (단순) 또는 "왜 RNN이 졌는가" (의문)

### Bullet

**선호**:
- 동사형 OK ("계산한다", "비교한다")
- 짧고 직관적 ("Q × K → 가중치")
- 예시 함께 ("예: 'I love you' → 3개 임베딩")

**회피**:
- 너무 압축 (학술 명사구만)
- 마침표 (강의는 호흡 단위)

### 인용 / 예시

강의 자료는 **구체적 예시**가 학술보다 훨씬 중요:
- 코드 스니펫 OK
- 실제 사용 사례 OK
- 비유 OK ("Self-Attention은 마치 ~처럼")

---

## 청중별 톤 조정

### 후배 / 학부생 대상

- 더 캐주얼, 비유 많이
- 배경 지식 가정 적게
- 예: "이게 처음에는 헷갈리는데, 사실 ~"

### 동료 / 대학원생

- 약어 / 전문 용어 OK
- 깊이 있게, but 학술 톤보다는 가벼움
- 예: "BERT가 결국 ~"

### 일반 청중 / 블로그

- 비유 / 일상 예시 강화
- 코드보다 그림으로
- 약어 풀어쓰기

---

## 인용 / 출처

학술과 달리 강의에서는:
- 모든 문장에 출처 X (당연한 사실은 생략 OK)
- 주요 인용만 명시 ("Vaswani et al., 2017이 제안")
- 외부 링크 (블로그, 영상)도 OK

slide 8 (takeaway)에 "Further reading" 섹션 추가도 OK:
```html
<div class="slide-footer">
  <div>다음: Multi-Head Attention</div>
  <div>참고: arxiv.org/abs/1706.03762</div>
</div>
```

## 구조 / 흐름

### 강의의 narrative arc

학술과 다른 패턴:
1. **Hook** — 호기심 자극 (질문, 놀라운 사실, 비유)
2. **Why** — 왜 이걸 배워야 하나
3. **What** — 핵심 개념 (분해해서 설명)
4. **How** — 동작 원리 / 사용법
5. **Takeaway** — 핵심 요약 + 다음 단계

학술의 "Background → RQ → Method → Result" 패턴과 다름.

### 호흡 단위 (paragraph break)

강의 자료는 청중의 인지 부하를 고려:
- 슬라이드당 7줄 이하 (P0-3 룰)
- 한 슬라이드에 한 개념 (compound idea X)
- 빠른 핸드오프 (다음 슬라이드 예고)

---

## 영어 / 한국어 혼용

학술과 비슷하지만 더 유연:

**원어 유지**:
- 학술 용어 (self-attention, embedding)
- 라이브러리 / 프레임워크 (PyTorch, TensorFlow)
- 모델 이름 (BERT, GPT-4)

**한국어로**:
- 일반 개념 (정확도, 성능, 비교)
- 강의 호흡어 ("그래서", "즉", "예를 들면")

**예시**:
- ✅ "Self-attention을 이해하면 BERT가 보인다"
- ✅ "Q와 K의 dot product는 유사도를 계산한다"
- ❌ "We will examine the self-attention" (영어/한국어 섞임)

## 시각 요소와 톤의 일치

테마별 톤 권장:

| 테마 | 어울리는 톤 |
|:---|:---|
| **light** | 차분, 정보 전달 ("핵심 정리", "단계별 이해") |
| **dark** | 임팩트, 깊이 ("왜 ~인가", "근본부터") |
| **paper** | 따뜻, 인문학 ("우리가 살펴볼", "여정") |

---

## 흔한 실수

1. **학술 톤 그대로 사용** — 딱딱해서 청중 흥미 ↓
2. **너무 캐주얼** — 슬라이드 본문은 여전히 정돈된 톤 ("ㅋㅋ", 이모지 ❌)
3. **호흡 단위 무시** — 한 슬라이드에 너무 많은 개념
4. **예시 없음** — 추상적 설명만, 구체적 예시 부족
5. **CTA 없음** — 마지막 슬라이드에 "다음 단계" 없음

---

## 좋은 강의 자료의 특징

체크리스트:
- [ ] 첫 슬라이드가 청중의 흥미를 끄는가 (Hook)
- [ ] 왜 배워야 하는지 명확한가 (Why)
- [ ] 핵심 개념을 한 슬라이드에 압축했는가
- [ ] 구체적 예시가 최소 1개 있는가
- [ ] 마지막 슬라이드에 다음 단계 / 추가 자료가 있는가
- [ ] 청중이 받아갈 takeaway 3개가 명확한가
